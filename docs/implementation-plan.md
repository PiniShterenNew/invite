# Implementation Plan & Completion Status — "באים?"

## שלבי הביצוע (כפי שבוצעו בפועל)
1. ביקורת החלטות ותיעוד שינויים (`product-decisions.md`) — לפני שורת קוד אחת.
2. אפיון: `product-brief.md`, `technical-architecture.md`, ואחריהם שאר `docs/*`.
3. סכמת נתונים (Prisma) + מיגרציה ראשונה.
4. שכבת `lib`: auth, authz, tokens, i18n, validation, rate-limit, storage, email, services (rsvp, stats, lifecycle, notify, access, event-view, guest-parse).
5. Design system (tokens, רכיבי UI בסיסיים, 4 תבניות עמוד אירוע).
6. חוויית אורח מלאה: הזמנה אישית/כללית, RSVP, פרופיל, רשימת משתתפים, קוד גישה.
7. אזור מארגן מלא: אשף יצירה בן 5 שלבים, דשבורד, ניהול מוזמנים ושיתוף, תצוגה מקדימה, ייצוא CSV.
8. עמודים ציבוריים (בית, מדיניות, תנאים, דיווח), התחברות, Admin מינימלי.
9. מייל (Resend + dev fallback), cron למחזור חיים (סיום אוטומטי, אזהרת מחיקה, אנונימיזציה, עיבוד תור מיילים).
10. Seed data ריאליסטי (6 אירועים, כל סטטוסי RSVP, כל רמות פרופיל, כל מצבי גישה).
11. בדיקות יחידה+אינטגרציה (Vitest, מול SQLite אמיתי).
12. `lint` + `typecheck` + `build` ירוקים; smoke test ידני מול שרת רץ אמיתי (לא רק קומפילציה).

## פערים פתוחים (לא מוסתרים)
1. **E2E (Playwright) לא נכתב** — התלות מותקנת, התרחישים מתועדים ב-testing-plan.md, אך המימוש בפועל דורש זמן נוסף שהוקצה במקום לכיסוי אינטגרציה על הלוגיקה העסקית הקריטית (RSVP concurrency, פרטיות, אנונימיזציה).
2. **אחסון S3 בפרודקשן** — הקוד מוכן מבנית (`lib/storage.ts`) אך מסלול ההעלאה בפועל ל-bucket חיצוני אינו ממומש (`throw UploadError("s3-not-configured-in-dev")`); דיסק מקומי עובד לגמרי לפיתוח/דמו.
3. **בדיקה ויזואלית אמיתית** (מכשיר פיזי, Safari, קורא מסך) לא בוצעה בסבב הזה — רק smoke test פונקציונלי מול שרת רץ.
4. **חסרי credentials חיצוניים** (Google OAuth, Resend, Postgres מנוהל) — לא הומצאו, לא זויפו. `.env.example` מפרט בדיוק מה חסר ומאיפה להשיג.

## מה כן נבדק בפועל (לא רק "אמור לעבוד")
- `npm run typecheck`, `npm run lint`, `npm run build` — כולם רצו ועברו נקי בסבב הזה.
- `npm test` — 37 בדיקות עברו, כולל בדיקת מרוץ אמיתית (concurrent RSVP submissions) מול SQLite אמיתי.
- שרת dev הורץ בפועל ו-smoke-tested עם `curl`: עמוד בית, קישור כללי (מציג את שם האירוע האמיתי מה-seed), קישור אישי (מציג כתובת מדויקת רק לאורח שכבר אישר YES — נבדקה חשיפת הכתובת המדורגת בפועל, לא רק בקוד), אירוע `PERSONAL_ONLY` חוסם גישה דרך קישור כללי, אירוע מוגן-קוד מציג את מסך הקוד, טוקן מזויף מחזיר "לא נמצא" עקבי, `/admin` ללא התחברות מפנה ל-login (307), ייצוא CSV ללא הרשאה מחזיר 401, `robots.txt` חוסם הכל.
